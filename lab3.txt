Реализовать мандатную систему защиты:
Должна появиться команда дозаписи в конец файла, например append. DONE

чем больше метка, тем прав больше
append: разрешен, если метка юзера меньше, либо равна метке объекта(файла либо директории)
write: разрешен, если метка юзера равна метке объекта
read: разрешен, если метка юзера выше либо равна метке объекта

Сервер:
- для пользователя создается метка, с которой он заходит на сервер (пользователь может определять метку, с которой он заходит, например, он может зайти с меткой ниже (но не выше) своей, если ему угодно)
- метка сессии должна определять права пользователя на сервере вместе с дискреционной моделью (работает по принципу приоритетного запрета, если в какой-то системе защиты есть запрет, то в итоге запрет, первой проверяется дискреционная система) (мандатная система должна обеспечить выполнение *-свойства) 
- в случае отказа в какой-либо системе пользователь должен понять, какая система ему отказала
- для каждого объекта и субъекта организовать хранение меток 	DONE
- команда для просмотра текущей метки (для пользователя/файла). 
Например: cm (current mark) -> unsigned int	DONE

Клиент: 
- интерфейс ввода метки, с которой он заходит на сервер	DONE

Схема работы 2 моделей:

Client			Server

read file -------------->
			проверка дискреционной системы
			(отправить отказ в случае запрета)
			проверка мандатной системы
отказ\ответ на запрос  <-
