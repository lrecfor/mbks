Реализовать механизм групп пользователей в виде команд для администратора.
Администратор должен:
- уметь создавать группы; DONE
- добавлять пользователей в группу; DONE
- удалять пользователей из группы; DONE
- удалять группу (не удаляя пользователей, входящих в эту группу). DONE
Для хранения групп и входящих в них пользователей можно использовать отдельный файл.

Реализовать дискреционную модель защиты:
В дискреционной модели защиты основной объект - матрица прав. DONE
Выглядит она следующим образом:

_\|s1 |s2 |...|sN
o1|r11|r21|r.1|rN1
o2|r12|r22|r.2|rN2
..|r1.|r2.|r..|rN.
oN|r1N|r2N|r.N|rNN, 

где
oN - объекты матрицы - файлы, папки и прочее,
sN - субъекты матрицы - пользователи и группы
rNM - право субъекта N на объект M. 
В такой матрице удобно хранить права в виде числа, составленного из битов прав

Требования:
Сервер:
ВАЖНОЕ ЗАМЕЧЕНИЕ: теперь клиентам должны быть доступны все директории сервера (в зависимости от установленных прав)! 
Реализовать описанную модель защиты:
- при каждом действии пользователя сначала проверяются соответствующие права на это действие:
	работает это по принципу приоритетного запрета, см. таблицу (u - пользователь, g - группа)
	u+ g+ -> OK
	u- g+ -> FORB
	u+ g- -> FORB
	u- g- -> FORB
	исходя из этого понятно, что любой запрет означает невозможность выполнить данное действие
	если какое-то право не установлено, то это тоже необходимо принять за запрет
- хранение в оперативной памяти описанной матрицы в любом удобном виде, к которой будет происходить обращение для всех проверок
- периодическая выгрузка матрицы в файл
- при удалении пользователя\группы также должны удаляться записи из матрицы, связанные с этим пользователем\группой
Добавить следующие команды для пользователя, обрабатываемые на сервере:
- чтение прав файла, допустим, rr filename DONE
- изменение прав файла: chmod filename u|g subjectName rights
